# One-Pager: UI Refinements (v3.2.1)

## Goal
Improve the clarity of the result card by prioritizing the User's Event Title and relocating the Samvatsara information.

## Proposed Changes

### 1. Result Card Header
*   **Current**: Displays the calculated *Samvatsara* (Year Name) as the main heading (e.g., "Krodhi").
*   **New**: Display the **Event Title** provided by the user in the input form (e.g., "My Birthday", "Diwali Puja").
    *   *Why*: Provides better context and personalization for the user.

### 2. Samvatsara Display
*   **Current**: Main heading.
*   **New**: Relocated to the results grid, specifically next to or grouped with **Vara** (Weekday).
    *   *Format*: A new grid item or combined with Vara (e.g., "Vara & Samvatsara").
    *   *Grid Update*: Add a dedicated row/cell for "Samvatsara" to ensure it remains prominent but distinct from the event title.

## Security Enhancement: Custom HTTPS Port
*   **Goal**: Run HTTPS on a random high port to evade standard scans.
*   **Port Selection**: **54321** (Example random high port).
*   **Implementation**:
    *   Update Nginx to listen on `54321 ssl`.
    *   Update Firewall rules to allow `54321/tcp`.
    *   Update `deploy.sh` to configure SELinux `http_port_t` for `54321`.

## Implementation Steps
1.  **Frontend (`index.html`)**:
    *   Change the `h2` ID in the result header to `res-title`.
    *   Add a new grid item `<div class="grid-item">...</div>` for **Samvatsara**.
2.  **Logic (`main.js`)**:
    *   Update `renderResult` function to:
        *   Set the text of the header (`res-title`) to the input title value.
        *   Set the text of the new Samvatsara grid item (`res-samvatsara`) from the API response.
3.  **Deployment (`deploy.sh` & templates)**:
    *   Modify `panchanga.nginx.template` to listen on `54321`.
    *   Add `semanage port -a -t http_port_t -p tcp 54321` to `deploy.sh`.
    *   Add `firewall-cmd --add-port=54321/tcp`.
